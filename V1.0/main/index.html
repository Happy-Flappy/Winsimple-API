    Linking

    -lgdi32 -luser32 -lmsimg32 -lkernel32

    Utility Functions

    WIDE(std::string str)

    Converts a UTF-8 string to a wide string. - Parameters: str - UTF-8 string to convert - Returns: std::wstring - Converted wide string

    TO_LPCSTR(std::string &str)

    Converts a C++ string to Windows LPCSTR type. - Parameters: str - String to convert - Returns: LPCSTR - Converted string for Windows API

    Class: Texture

    Public Variables

    • HBITMAP bitmap - Windows bitmap handle
    • int width - Texture width in pixels
    • int height - Texture height in pixels
    • COLORREF transparencyColor - Color to treat as transparent (default: CLR_INVALID)

    Constructors

    • Texture() - Default constructor
    • Texture(std::string path) - Constructor that loads texture from file

    Public Methods

    • bool load(std::string path) - Loads bitmap from file
    • Parameters: path - Path to image file

    • Returns: true if successful, false otherwise

    • bool isValid() const - Checks if texture is loaded

    • Returns: true if bitmap is loaded, false otherwise

    • void setTransparentMask(COLORREF color) - Sets color to treat as transparent

    • Parameters: color - RGB color to mask as transparent

    Class: Shape

    Public Variables

    • int x, y, z - Position coordinates
    • float scaleX, scaleY - Scale factors (default: 1.0)
    • int originX, originY - Origin point for transformations (default: 0)
    • COLORREF color - Fill color (default: white)
    • RECT rect - Bounding rectangle and texture source rectangle

    Constructors

    • Shape() - Default constructor (10x10 white rectangle at origin)

    Public Methods

    • bool contains(int posx, int posy) - Checks if point is inside shape
    • Parameters: posx, posy - Point coordinates to test

    • Returns: true if point is inside shape

    • void setTexture(Texture &texture) - Assigns texture to shape

    • Parameters: texture - Reference to Texture object

    • Texture &getTexture() - Gets the shape's texture

    • Returns: Reference to assigned Texture

    • const Texture* getTexture() const - Gets the shape's texture (const version)

    • bool hasTexture() const - Checks if shape has texture assigned

    • Returns: true if texture is assigned

    • void fitTexture() - Resizes shape to match texture dimensions

    • void setOrigin(int posx, int posy) - Sets the transformation origin point

    • Parameters: posx, posy - Origin coordinates

    Class: Window

    Public Variables

    • HWND hwnd - Window handle
    • int x, y, width, height - Window position and dimensions
    • View view - Viewport configuration structure containing:
    • int x, y, width, height - Viewport position and size
    • World world - World space configuration:
      • int x, y, width, height - World space coordinates

    Constructors

    • Window(int width, int height, std::string title) - Creates a window
    • Parameters:
      • width, height - Window dimensions
      • title - Window title

    Public Methods

    • bool isOpen() - Main window loop, processes messages

    • Returns: true while window is open

    • bool pollEvent(MSG &message) - Retrieves next event from queue

    • Parameters: message - Reference to MSG structure to fill

    • Returns: true if event was retrieved

    • void clear(COLORREF color = RGB(255, 255, 255)) - Clears back buffer

    • Parameters: color - Clear color (default: white)

    • void draw(Shape &shape) - Draws a shape to back buffer

    • Parameters: shape - Shape to draw

    • void display() - Presents back buffer to screen

    • void setFullscreen(bool fullscreen) - Toggles fullscreen mode

    • Parameters: fullscreen - true for fullscreen, false for windowed

    • void toggleFullscreen() - Toggles between windowed and fullscreen

    • bool getFullscreen() const - Gets current fullscreen state

    • Returns: true if in fullscreen mode

    Notes

    • Window class uses a double buffering system with back buffer
    • Shapes with textures are drawn with transparency support
    • World/view system allows for viewport management
    • Automatic resource cleanup on destruction
